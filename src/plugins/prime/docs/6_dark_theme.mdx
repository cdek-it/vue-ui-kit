# Тёмная тема

## Текущий статус

Тёмная тема **отключена** в конфигурации DS. В `getPrimeVueConfig()` установлено `darkModeSelector: false`.

## Что уже готово

Токены дизайн-системы (`tokens.json`) содержат полную секцию `colorScheme.dark` — все семантические цвета (surface, primary, text, severity) определены для тёмной схемы.

Пример: в тёмной теме шкала Surface инвертируется — `surface-50` становится тёмным (`zinc.950`), `surface-900` — светлым (`zinc.50`).

## Что нужно для включения

1. **Установить `darkModeSelector`** в конфигурации PrimeVue:

```js dark
// Вариант 1: CSS-класс (управление вручную)
app.use(PrimeVue, getPrimeVueConfig({
  theme: {
    options: {
      darkModeSelector: '.dark',
    },
  },
}));

// Вариант 2: системные настройки
app.use(PrimeVue, getPrimeVueConfig({
  theme: {
    options: {
      darkModeSelector: 'prefers-color-scheme: dark',
    },
  },
}));
```

2. **Проверить компоненты** — убедиться, что все состояния корректно отображаются в тёмной теме.

3. **Проверить кастомные стили** — если в проекте есть стили с захардкоженными цветами (не через токены), они не переключатся автоматически.

## Известные ограничения

- Макетов с тёмной темой нет — некоторые состояния компонентов могут отображаться некорректно
- Типографика (`typography.min.css`) не имеет dark-варианта — потребуется отдельная работа
- При обнаружении ошибок создавайте issue в репозитории
